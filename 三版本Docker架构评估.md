# 三版本 Docker 架构评估报告

> 评估对象：用户建议用三个 Docker 容器分别部署通用版、幼升小版、小升初版
> 评估维度：技术、产品、运营、成本
> 日期：2026-04-21

---

## 一、建议回顾

用户建议：
- 通用版 Docker：覆盖全年级全学科
- 幼升小版 Docker：专注入学准备
- 小升初版 Docker：专注升学冲刺

---

## 二、技术维度评估

### ✅ 合理之处

| 点 | 说明 |
|:---|:---|
| 代码隔离 | 三个版本 prompt、报告模板、数据库完全独立，互不干扰 |
| 独立迭代 | 可以分别上线、分别回滚、分别优化 |
| 故障隔离 | 一个版本崩了，另外两个不受影响 |

### ❌ 不合理之处

| 点 | 说明 |
|:---|:---|
| **服务器资源吃紧** | 当前阿里云 2核4G，一个 Next.js 占 200-400MB，三个容器 600MB-1.2GB，加上系统开销，**4G内存几乎没有余量** |
| **数据库不互通** | 三个版本 = 三个 SQLite 数据库，孩子升学了数据无法迁移 |
| **运维成本翻倍** | 部署、监控、日志、备份都要做三份 |
| **证书/域名** | 三个子域名都需要 SSL 证书和备案解析 |

**技术结论：方向对，但三个容器太重。**

---

## 三、产品维度评估

### ✅ 合理之处

| 点 | 说明 |
|:---|:---|
| 定位清晰 | 每个版本有专属感，家长不会觉得"四不像" |
| 差异化定价 | 幼升小版可以定¥199，小升初版定¥299 |
| 精准营销 | "幼升小必备"、"小升初冲刺"可以分别打关键词 |

### ❌ 不合理之处

| 点 | 说明 |
|:---|:---|
| **用户分流混乱** | 家长拍作业前要先选"我家是幼升小还是小升初"，多一步操作 = 多一步流失 |
| **升学数据断层** | 孩子在幼升小版用了一年，上一年级了，数据怎么办？重新注册？ |
| **多孩家庭灾难** | 三个孩子三个版本，家长要记三个网址、付三笔钱 |
| **中间年级被遗忘** | 二三四年级孩子家长进哪个版本？通用版？那通用版和垂直版的区别是什么？ |

**产品结论：垂直定位对，但物理隔离带来用户体验问题。**

---

## 四、运营维度评估

### ✅ 合理之处

- 可以分别做投放、分别看转化数据
- 幼升小和小升初的获客渠道完全不同（幼儿园家长群 vs 小升初论坛）

### ❌ 不合理之处

| 点 | 说明 |
|:---|:---|
| **三倍备案** | wingo.asia 只有一个，子域名也要备案，时间成本 |
| **三倍客服** | "我买的幼升小版能看小升初吗？"这类问题会暴增 |
| **定价混乱** | 三个版本三个价格，家长比价后可能选最便宜的，不管适不适合 |

---

## 五、成本维度评估

| 方案 | 月成本 | 说明 |
|:---|---:|:---|
| 当前（单容器） | ¥1,415 | 2核4G + 域名 + AI包月 |
| 三容器（共用一台） | ¥1,415 | 4G内存吃紧，有性能风险 |
| 三容器（各一台） | ¥3,045 | 三台2核4G，成本翻倍 |

**成本结论：当前阶段扛不住三容器独立部署。**

---

## 六、综合评估结论

### 用户的方向是对的，但实现方式可以优化

用户想达到的**本质目标**：
1. 不同年级的孩子得到不同的分析逻辑
2. 不同阶段的家长看到不同的定价和界面
3. 产品定位清晰，不四不像

**这些目标不需要三个 Docker 也能实现。**

---

## 七、替代方案推荐

### 方案：一套代码 + 年级动态切换（推荐）

**实现方式：**
- 同一个 Next.js 应用，同一个 Docker 容器
- 拍作业后，AI 自动判断年级（幼升小/小学低/小学高/初中）
- 根据年级自动加载对应的 prompt 和报告模板
- 数据库按 `user_id + child_id + grade` 存储，数据互通

**产品体验：**
- 家长拍作业 → 系统自动识别"这是幼升小口算"或"这是小升初几何"
- 一个账号可以添加多个孩子（老大幼升小、老二小升初）
- 孩子升学后，历史数据自动延续，不用换版本

**成本：**
- 还是单容器，¥1,415/月不变

**效果等价性：**
- 家长感知：拍幼升小作业 → 看到幼升小风格的报告
- 家长感知：拍小升初作业 → 看到小升初风格的报告
- 和三版本 Docker 的效果**几乎一样**，但成本低、体验好

---

## 八、一句话结论

> **三版本 Docker 的思路对（垂直定位），但实现方式重（资源/运维/用户体验）。建议用"一套代码 + 年级动态切换"达到同样效果，成本不变，体验更好。等某个版本（比如小升初）跑通了、量大了，再单独拆分出去。**
