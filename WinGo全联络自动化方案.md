## 三、自动化工作流设计（n8n）

### 3.1 核心工作流清单

```
Webhook: wingo-events
    │
    ├─ event = user_registered
    │   ├─ 发送欢迎模板消息（微信）
    │   ├─ 飞书通知运营
    │   └─ Dify生成个性化欢迎语（可选）
    │
    ├─ event = report_generated
    │   ├─ 发送报告模板消息（微信）
    │   ├─ 检查剩余次数（如果是最后1次，附加付费引导）
    │   ├─ 飞书通知运营
    │   └─ 存入数据库（报表统计）
    │
    ├─ event = free_quota_exhausted
    │   ├─ 发送付费引导模板消息（微信）
    │   ├─ 飞书通知运营（重点跟进标签）
    │   └─ 标记用户状态：待转化
    │
    ├─ event = payment_success
    │   ├─ 发送开通成功模板消息（微信）
    │   ├─ 飞书通知运营+销售
    │   ├─ 激活会员权益
    │   └─ 存入数据库（收入统计）
    │
    └─ event = share_report
        ├─ 增加免费次数（API调用）
        ├─ 发送奖励到账通知（微信）
        └─ 飞书通知运营（裂变数据）
```

### 3.2 定时工作流

```
每日 0:00
    ├─ 查询昨日数据
    ├─ 生成日报
    └─ 推送到飞书群

每周日 22:00
    ├─ 汇总本周数据
    ├─ 生成趋势图
    └─ 推送到飞书群

每月1日 9:00
    ├─ 查询机构分成数据
    ├─ 生成结算单
    ├─ 发送邮件给机构负责人
    └─ 飞书通知财务

每日 10:00
    ├─ 查询7天未活跃用户
    └─ 批量发送唤醒模板消息

会员到期前7天
    ├─ 查询即将到期用户
    └─ 批量发送续费提醒
```

## 五、数据闭环

### 5.1 每个联络节点都记录

```sql
-- 联络记录表
CREATE TABLE contact_logs (
  id TEXT PRIMARY KEY,
  user_id TEXT,
  contact_type TEXT,      -- wechat_template / feishu / sms / email
  scene TEXT,             -- report_generated / quota_warning / payment_success
  content TEXT,           -- 发送内容摘要
  status TEXT,            -- sent / failed / clicked
  sent_at TEXT,
  clicked_at TEXT,        -- 用户点击时间（如有链接）
  created_at TEXT
);
```

### 5.2 效果追踪

| 指标 | 计算方式 |
|------|---------|
| 模板消息打开率 | 点击数 / 发送数 |
| 付费引导转化率 | 付费人数 / 引导推送人数 |
| 唤醒成功率 | 唤醒后7天内使用人数 / 唤醒人数 |
| 裂变系数 | 分享带来的新用户 / 分享人数 |

## 六、异常处理

### 6.1 微信模板消息被封

**备用通道**：
1. 公众号客服消息（48小时内可发）
2. 公众号图文消息（订阅号每天1次）
3. 企业微信（加家长企微）

### 6.2 n8n 宕机

**降级方案**：
- 后端事件中心直接调微信API（不经过n8n）
- 关键通知（付费、告警）走备用通道

### 6.3 消息频繁被封

**频率控制**：
- 同一用户每天最多3条模板消息
- 优先发送高价值消息（报告生成、付费引导）
- 低优先级消息合并发送

## 七、实施步骤

| 阶段 | 时间 | 目标 |
|------|------|------|
| **Phase 1** | 本周 | 跑通基础链路：注册→报告→推送→飞书通知 |
| **Phase 2** | 下周 | 增加转化链路：限额提醒→付费引导→开通通知 |
| **Phase 3** | 第3周 | 增加留存链路：7天唤醒→续费提醒→会员到期 |
| **Phase 4** | 第4周 | 增加B端链路：机构入驻→班级报告→结算通知 |
| **Phase 5** | 第2月 | 数据看板：自动化日报+周报+异常告警 |

---

**核心原则**：
1. **全自动** — 从触发到触达，零人工
2. **全渠道** — 微信+飞书+邮件+短信，该用什么用什么
3. **全记录** — 每个联络节点都留痕，可追溯
4. **可降级** — 主链路挂了，备用链路自动顶上
