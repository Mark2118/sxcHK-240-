## 二、竞争格局（谁在场上）

### 第一梯队（占据82.3%学习机市场）
- **作业帮** — 学习机+AI对话+作文批改+知识点分析
- **科大讯飞** — 星火大模型+AI作文批改+智能组卷+学情诊断
- **学而思（好未来）** — 九章大模型+AI老师
- **步步高** — Deepseek X AI老师
- **小猿（猿辅导）** — 拍照搜题+AI批改

### 其他玩家
- **笔神作文批改** — 学情分析模块（班级作业分析报告、能力统计）
- **天学网** — AI批改系统（已通过网信办教育场景安全评估备案）
- **一起作业/校宝** — 公立学校SaaS（教务管理、作业记录）

### 关键发现
> **现有产品全部聚焦：公立学校、大型机构、C端家长（学习机）**
> 
> **私立中小型教育机构市场 = 空白**

## 三、政策环境（合规红线）

| 政策 | 影响 |
|------|------|
| **双减** | 学科类线下/线上授课被强监管，供给侧收缩 |
| **AI教育工具安全评估** | 网信办"教育场景"专项评估，违规=下架+罚款 |
| **合规形态** | AI以"工具与内容"切入，不直接授课，成为合规补充 |

**结论**: 产品定位必须是"学情分析工具"，绝不能碰"授课/辅导"，文案上规避培训暗示。

## 四、用户痛点（从调研中提取）

### C端家长
- 作业辅导刚需仍在，但校外辅导减少
- 需要"低成本、可持续"的辅导工具
- 学习机（2000-10000元）门槛高，且被投诉"智商税"
- 学情诊断功能受欢迎："几秒出报告，精准巩固薄弱点"

### B端私立机构
- **掌心宝贝** — 做记录，无分析
- **校宝** — 做教务，无学情
- **现有AI批改** — 全部面向公立学校/大机构，私立小机构用不起/买不到
- **核心痛点**: 无法向家长证明"孩子进步了"（教学效果不可视化）

## 五、技术栈验证（n8n + Dify + OpenMAIC）

### 行业落地案例
- **2026年1月** — BuildingAI + LangChain + n8n + Dify 方案已被验证可1周内落地商用AI平台
- **Dify** — 教育场景知识库问答（某在线教育平台用Dify做AI助教，学生参与度+45%）
- **n8n** — 跨平台自动化（客服→CRM→邮件→通知全链路）

### 选型结论
PRD v6 的 XSC+n8n+Dify+OpenMAIC 生态架构 **符合行业趋势**，不是凭空想象。

## 六、产品定位建议（供架构图参考）

### 一句话定位
> **私立教育机构的 AI 学情分析引擎 — 不替代现有系统，只补分析缺口。**

### 与竞品差异

| 维度 | 作业帮/科大讯飞 | WinGo |
|------|---------------|-------|
| **目标用户** | C端家长（学习机） | **B端私立机构 + C端家长（通过机构）** |
| **核心功能** | 拍照搜题/讲课 | **学情诊断+薄弱点追踪+行动清单** |
| **使用场景** | 家庭自学 | **机构教学效果可视化** |
| **商业模式** | 硬件+会员 | **SaaS订阅+按分析次数计费** |
| **差异化** | 大而全 | **小而深：只做学情分析，不做授课** |

### 三个"竞品空白"（PRD v6 已识别，调研确认）

1. **今晚行动清单** — 报告直接转化为"今晚该辅导什么"（调研中家长明确需求"精准巩固"）
2. **薄弱点跨时间追踪** — 追踪30天，看进步还是退步（现有产品都是单次分析，无时间线）
3. **教学效果可视化** — 机构向家长证明"孩子进步了"（私立机构核心痛点，现有工具无法提供）

## 七、风险预判

| 风险 | 概率 | 应对 |
|------|------|------|
| **大厂跟进**（作业帮/科大讯飞推出机构版） | 中 | 速度取胜，5周内上线；深度绑定私立机构场景 |
| **政策收紧**（学情分析也被纳入监管） | 低 | 坚持"工具"定位，不碰授课；文案合规 |
| **AI幻觉导致报告错误** | 中 | 百度客观批改+AI只讲评（对错分离） |
| **家长付费意愿低** | 中 | B端机构付费为主，C端免费3次转化 |
| **n8n/Dify稳定性** | 低 | Docker部署+自托管，不依赖云服务商 |

## 八、给架构图的关键输入

### 必须画进去的核心模块
1. **XSC 核心引擎** — 拍照→OCR→切题→批改→报告→行动清单（现有代码已有，需升级）
2. **薄弱点追踪引擎** — 时间线数据库+趋势计算（新建，竞品空白）
3. **n8n 自动化层** — 7条工作流（注册/推送/转化/留存/续费/唤醒/B端跟进）
4. **Dify 客服层** — L1问题自动回复（80%）
5. **OpenMAIC 咨询层** — L3深度咨询+B端销售演示
6. **B端管理后台** — 机构注册/学员管理/班级看板/品牌定制
7. **微信生态** — 扫码登录/支付/模板消息推送（C端入口）

### 数据流核心
```
家长扫码 → 微信授权 → 上传作业 → XSC分析 → 报告生成
                                          ↓
                                    n8n自动推送
                                          ↓
                              家长收到微信消息"报告已生成"
                                          ↓
                              点击查看 → 薄弱点+行动清单
                                          ↓
                              免费3次用完 → 付费弹窗 → 微信支付
```

### B端数据流
```
机构注册 → 导入学员 → 批量上传作业 → 批量分析 → 班级看板
                                              ↓
                                        薄弱点热力图+雷达图
                                              ↓
                                        每日简报自动推送给家长
                                              ↓
                                        报告上显示机构Logo（品牌定制）
```

## 九、关键数字（记到架构图上）

- **目标**: 一人运营 1000+ C端用户 + 50+ B端机构
- **C端转化**: 免费3次 → 付费（月卡¥29.9 / 年卡¥239）
- **B端收费**: SaaS订阅 + 按分析次数计费

- **自动化率**: 80%日常工作自动化
- **客服分流**: L1(80%)→Dify / L2(15%)→Dify+n8n / L3(5%)→OpenMAIC
