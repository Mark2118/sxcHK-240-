### 1.2 文档格式规范

每篇知识库文档包含：
- **问题标题**：用户可能问的问题
- **关键词**：用于检索匹配
- **回答正文**：结构化回答
- **相关链接**：引导用户查看更多

示例：
```markdown
# 如何拍照上传

## 关键词
拍照、上传、照片、图片、作业、怎么传

## 回答
上传作业照片很简单，只需要3步：

1. 点击首页的"拍照上传"按钮
2. 选择"相机拍摄"或"相册选择"
3. 确认照片清晰后，点击"开始分析"

**拍照小贴士：**
- 确保光线充足
- 文字清晰可见
- 尽量正对着作业拍摄
- 如果有多页，可以批量上传（最多5页）

**常见问题：**
- Q: 照片上传失败怎么办？
  A: 检查网络连接，或尝试压缩图片后重新上传
- Q: 可以上传手写体吗？
  A: 可以，但印刷体识别准确率更高

[查看详细教程](https://wingo.asia/guide/photo)
```

## 二、对话场景设计

### 2.1 场景矩阵

| 场景 | 触发条件 | 处理方式 | 转人工条件 |
|------|---------|---------|-----------|
| 使用指导 | 包含"怎么"/"如何"/"怎么看" | Dify知识库回答 | 连续3次未解决 |
| 会员咨询 | 包含"会员"/"价格"/"多少钱" | Dify知识库+引导付费 | 要求退款/投诉 |
| 问题排查 | 包含"失败"/"错误"/"用不了" | Dify排查+API查询 | API查询失败 |
| 学情解读 | 包含"什么意思"/"看不懂" | Dify解释+图文 | 要求深度规划 |
| B端咨询 | 包含"机构"/"入驻"/"合作" | Dify回答+引导预约 | 要求立即演示 |
| 投诉建议 | 包含"投诉"/"不好"/"退款" | Dify安抚+转人工 | 立即转人工 |

## 三、API集成

### 3.1 Dify → XSC 查询接口

```typescript
// Dify 调用 XSC API 获取用户实时状态

// 查询用户会员状态
GET /api/user/status?openid={openid}
Response: {
  membership: "free",
  freeCountRemaining: 0,
  freeCountLimit: 3,
  expiresAt: null
}

// 查询报告摘要
GET /api/report/{reportId}/summary
Response: {
  overallScore: 78,
  weakPoints: ["计算", "应用题"],
  actionList: ["练习分配律", "巩固几何公式"]
}

// 查询学情趋势摘要
GET /api/trends/summary?userId={userId}&period=week
Response: {
  improvement: "+8%",
  bestTopic: "计算模块",
  worstTopic: "应用题"
}
```

### 3.2 Dify 配置

```yaml
# Dify 应用配置
app:
  name: "WinGo 智能客服"
  description: "WinGo学情管家官方智能客服"
  mode: "chat"
  
# 模型配置
model:
  provider: "minimax"  # 或 openai/deepseek
  model: "abab6.5"
  temperature: 0.7
  max_tokens: 1000

# 知识库配置
knowledge:
  - name: "使用指南"
    description: "产品使用教程"
    retrieval_mode: "semantic"
    top_k: 5
  - name: "会员权益"
    description: "会员相关问答"
    retrieval_mode: "semantic"
    top_k: 3
  - name: "学情科普"
    description: "学情分析知识"
    retrieval_mode: "semantic"
    top_k: 3

# 工具配置
tools:
  - name: "query_user_status"
    description: "查询用户会员状态"
    endpoint: "http://100.106.90.55:3002/api/user/status"
  - name: "query_report_summary"
    description: "查询报告摘要"
    endpoint: "http://100.106.90.55:3002/api/report/{id}/summary"
  - name: "query_trends"
    description: "查询学情趋势"
    endpoint: "http://100.106.90.55:3002/api/trends/summary"

# 开场白
opening_statement: |
  您好！我是 WinGo 智能客服助手 🤖
  
  我可以帮您：
  • 解答产品使用问题
  • 查询会员权益
  • 解释学情报告
  • 了解B端合作
  
  请问有什么可以帮您的？

# 转人工规则
escalation:
  - condition: "用户连续3次表示未解决问题"
    action: "转人工客服"
  - condition: "用户要求退款/投诉"
    action: "立即转人工"
  - condition: "用户要求深度咨询（学习规划/报班建议）"
    action: "引导至 OpenMAIC 深度咨询"
```

## 四、效果评估

### 4.1 关键指标

| 指标 | 目标 | 测量方式 |
|------|------|---------|
| 问题解答率 | ≥80% | Dify回答后用户未追问"转人工" |
| 平均响应时间 | <3秒 | Dify首次回复时间 |
| 用户满意度 | ≥4.5/5 | 对话结束后评分 |
| 转人工率 | <15% | 转人工对话占比 |
| 付费引导转化率 | ≥10% | Dify引导后用户付费比例 |
