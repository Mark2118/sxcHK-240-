## 二、各组件定位

| 组件 | 定位 | 核心能力 | 不做什么 |
|------|------|---------|---------|
| **XSC** | 核心业务引擎 | 拍照分析、学情报告、行动清单、薄弱点追踪 | 不做客服、不做营销 |
| **n8n** | 自动化引擎 | 工作流编排、定时任务、数据同步、通知推送 | 不做AI对话、不做业务逻辑 |
| **Dify** | 智能客服 | 常见问题自动回复、知识库问答、使用引导 | 不做深度咨询、不做销售 |
| **OpenMAIC** | 深度咨询 | 学情解读、报班建议、学习规划、B端销售演示 | 不做简单FAQ（留给Dify） |

### 分工原则

```
用户问题分级：

L1 简单问题（80%）→ Dify 自动处理
  ├── "怎么看报告？"
  ├── "免费额度用完了怎么办？"
  ├── "怎么上传照片？"
  └── "会员有什么权益？"

L2 中度问题（15%）→ Dify + n8n 联动
  ├── "我的报告为什么没生成？" → Dify 查询状态 → n8n 自动重试
  ├── "我想退款" → Dify 解释政策 → n8n 创建工单
  └── "报告数据不对" → Dify 收集信息 → n8n 通知人工

L3 深度问题（5%）→ OpenMAIC 处理
  ├── "孩子计算总是错，是不是该报班？"
  ├── "小升初该怎么准备？"
  ├── "你们产品怎么收费？"（B端销售咨询）
  └── "能给我们机构做个演示吗？"
```

## 三、n8n 工作流设计（自动化引擎）

### 3.1 新用户注册工作流

```
触发：XSC /api/auth/wechat 注册成功
  ↓
动作1：发送欢迎模板消息（微信）
  "欢迎加入 WinGo 学情管家！前3次分析免费，点击开始体验 →"
  ↓
动作2：延迟3天
  发送"第一次使用指南"
  "还没用过？3分钟教你看懂学情报告 → [图文教程]"
  ↓
动作3：延迟7天（如果未使用）
  发送"限时福利"
  "再送你2次免费分析，试试看 →"
  ↓
动作4：延迟14天（如果仍未付费）
  发送"案例分享"
  "看看其他家长怎么用 WinGo 帮孩子提升的 →"
```

### 3.2 报告生成推送工作流

```
触发：XSC /api/check 报告生成成功
  ↓
条件判断：
  ├── C端用户 → 调用微信模板消息 API
  │   "您孩子的学情报告已生成！薄弱点：计算模块"
  │   "点击查看完整报告 + 今晚行动清单 →"

  └── B端机构用户 → 同时推送
      ├── 推送给家长（个人报告）
      └── 推送给机构管理员（班级汇总简报）
```

## 五、OpenMAIC 深度咨询设计

### 5.1 C端深度咨询场景

**场景1：学情解读**
```
家长：孩子计算总是错，是不是该报班？
OpenMAIC：根据您孩子的学情数据，我为您分析一下：
  【现状】最近5次作业，计算模块错误率35%，主要问题是：
    - 分配律和结合律混淆（出现3次）
    - 进退位计算粗心（出现2次）
  【判断】这不是"不会"，是"概念不清+习惯问题"
  【建议】
    1. 先不报班：用 WinGo 生成的5道针对性练习巩固
    2. 观察2周：如果错误率降到15%以下，说明有效
    3. 如果持续高于25%：再考虑报计算专题班
  【行动】已为您生成专项练习，今晚开始 → [链接]
```

**场景2：学习规划**
```
家长：小升初该怎么准备？
OpenMAIC：基于您孩子目前的学情，我为您制定一个3个月规划：
  【当前水平】计算85%、应用题60%、几何70%
  【目标】小升初数学达到90%正确率
  【3个月计划】
    第1个月：应用题专项（薄弱点TOP1）
    第2个月：几何专项（薄弱点TOP2）
    第3个月：综合冲刺（模拟测试+查漏补缺）
  【每周安排】
    - 周一/三/五：上传作业，WinGo自动分析
    - 周二/四：薄弱点专项练习
    - 周六：错题回顾
    - 周日：休息
  【需要我每周提醒您执行吗？】
```
