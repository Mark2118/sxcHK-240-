# WinGo 学情洞察 — 产品深度思考：如何做到"点一下"

**核心问题**：家长从知道 WinGo 到获得学情报告，能不能只点一下？

**沿着您的问题，我的思考**：

---

## 一、"点一下"的完整定义

### OpenClaw 为什么能做到"点一下"？

```
用户点击"领取"
    ↓
系统后台自动完成：部署、配置、启动
    ↓
用户拿到可用服务
```

**关键**：用户只做了一个动作（点一下），其余全是系统自动。

### WinGo 应该是什么样的"点一下"？

```
家长扫码（或点击链接）
    ↓
系统自动完成：授权、注册、进入分析页
    ↓
家长直接上传作业
    ↓
报告生成，微信推送
```

**家长感知到的动作**：
1. 扫码（1下）
2. 上传作业（1下）

**系统后台自动完成的**：
- 微信网页授权（拿openid）
- 用户注册（新用户自动创建）
- 进入 analyze 页面
- 报告生成后模板消息推送

---

## 二、授权流程怎么设计？

### 您的核心诉求：必须拿到注册信息（openid）

**没有 openid = 断联 = 无法后续触达 = 无法转化**

所以授权不能后置，必须在第一步完成。

### 但授权前置 ≠ 授权墙

**错误做法**（现在的落地页）：
```
打开页面 → 看到授权按钮 → 点击授权 → 跳转微信 → 授权成功 → 回到页面
```
- 家长感知：4步，每一步都在流失

**正确做法**（扫码链路）：
```
家长扫码 → 微信自动授权（无感知）→ 直接进入 analyze
```
- 家长感知：1步（扫码），后面全是自动

### 关键设计：二维码带场景参数

```
二维码内容：https://zjcz.top/xsc/analyze?scene=wx_qr&from=poster

后端逻辑：
1. 检测到 scene=wx_qr
2. 检查是否已登录（cookie）
3. 未登录 → 302 到微信授权页
4. 微信授权回调 → 自动注册/登录 → 重定向回 analyze
5. 家长感知：扫码后直接进入 analyze，无中断
```

### 如果直接点击链接（非扫码）怎么办？

**方案 A**：同样逻辑，点击链接 → 未登录 → 自动跳转授权 → 回来
**方案 B**：先展示极简落地页（1屏），页内按钮点击后走授权

**建议**：正式环境以扫码为主（幼儿园海报、朋友圈海报），落地页作为备用。

## 三、页面怎么做到极简？

### 当前问题：信息过载

落地页 9 个 section，analyze 功能繁杂。

### "点一下"的页面哲学

**OpenClaw 的页面逻辑**：
- 用户知道要干嘛（领取服务）
- 页面只有一个按钮（点一下）
- 没有解释、没有教育、没有说服

**WinGo 应该一样**：
- 家长扫码时，已经在海报/朋友圈知道了"这是学情诊断"
- 不需要再教育
- 页面只有一个动作：上传作业

### analyze 页面重构

```
┌─────────────────────────────┐
│  WinGo 学情洞察              │
├─────────────────────────────┤
│                              │
│    [ 点击拍照上传作业 ]       │  ← 首屏唯一核心按钮
│                              │
│    或粘贴文字到下方          │
│                              │
│    ┌─────────────────────┐  │
│    │                     │  │
│    │   文本输入框         │  │
│    │                     │  │
│    └─────────────────────┘  │
│                              │
│    [学科] 数学 ▼             │
│                              │
│    [ 开始分析 ]              │  ← 大按钮
│                              │
└─────────────────────────────┘
```

**删除**：
- 历史报告入口（推送到微信，点消息看）
- 会员中心（免费次数用完时自然弹出）
- 用户信息（顶部只显示头像，不显示次数）
- 复杂导航

**保留**：
- 上传/输入
- 学科选择
- 分析按钮
- 底部："剩余免费次数 X 次"（小字，不打扰）

## 四、报告页怎么设计？

### 当前问题：报告太复杂，家长看不懂

### "点一下"的报告逻辑

**家长打开报告时，3 秒内必须get到3个信息**：
1. 孩子整体掌握度（%）
2. 最危险的薄弱点（Top 1）
3. 今天该干什么（行动指令）

### 报告页结构

```
┌─────────────────────────────┐
│  ← 数学学情诊断              │
├─────────────────────────────┤
│                              │
│   掌握度 87%                 │
│   ████████████████████░░░   │
│                              │
│   ⚠️ 薄弱点：行程问题        │
│   本周出现4次，正确率25%     │
│                              │
│   💡 今日行动                │
│   练习10道相遇问题           │
│   [获取专项练习]             │
│                              │
├─────────────────────────────┤
│   📊 知识模块详情            │
│   [展开查看]                 │
├─────────────────────────────┤
│   📥 下载完整报告            │
└─────────────────────────────┘
```

**关键**：
- 首屏只给结论和行动，不给过程
- 详情折叠，需要再展开
- 下载按钮在底部，引导保存

## 五、WinGo 怎么表述才专业？

### 当前问题：像初创项目，不够专业

### 专业化的三层

**第一层：品牌名**
- 不用 "AI"、"智能" 这类烂大街的词
- 用 "诊断"、"评估"、"分析" 这类医疗/教育专业词
- 类比：不是 "AI 看病"，是 "影像诊断"

**第二层：功能名**
- "拍照批改" → "作业影像采集与学情评估"
- "分析报告" → "学情诊断报告"
- "薄弱点" → "知识掌握度缺口"

**第三层：文案语气**
- 不用感叹号
- 不用"哦""呢""吧"
- 用陈述句，数据说话

### 文案对照表

| 场景 | 现在（业余） | 专业版 |
|------|------------|--------|
| 首屏标题 | "AI 一眼看穿孩子的学习盲区" | "学情诊断，精准定位知识缺口" |
| 按钮 | "免费体验" | "开始诊断" |
| 报告标题 | "学情分析报告" | "数学学科诊断报告" |
| 薄弱点 | "行程问题没掌握" | "行程问题掌握度 25%，建议专项训练" |
| 推送消息 | "报告生成啦，快来看~" | "您孩子的数学诊断报告已生成" |

## 六、安全与防刷（已修，但需持续）

### 当前已做
- API 强制 token 鉴权（未登录无法调用）
- 免费次数限制（新用户默认 3 次）
- 每次调用扣减次数

### 还需做的
- 同一 openid 短时间内多次调用 → 限流
- 异常 IP 批量注册 → 封禁
- 报告生成频率监控 → 触发告警

## 七、推送怎么闭环？

### 当前状态
- 报告生成 → emitMarketingEvent → n8n → 微信模板消息
- n8n 01 已修好，链路待测试

### "点一下"的推送体验

```
家长上传作业（点一下）
    ↓
10秒后，微信收到推送
    ↓
点击推送 → 直接打开报告页面
    ↓
看完报告 → 底部提示"剩余X次"
```

**关键**：推送不是"通知"，是"交付"。家长不用记网址，点推送就看。

**核心结论**：

WinGo 的"点一下" = **家长只做两个动作：扫码 + 上传。其余全是系统自动。**

系统背后要完成的：
1. 授权无感（扫码即授权）
2. 注册自动（新用户静默创建）
3. 页面极简（只留上传）
4. 报告专业（数据说话）
5. 推送闭环（点推送就看）
